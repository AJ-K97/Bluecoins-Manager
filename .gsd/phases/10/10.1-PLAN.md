# Phase 10.1: Advanced Resource Management Plan

## Objective
Allow full administrative control over AI rulebooks, categories, and transactions through the Telegram bot interface. This reduces the need for CLI access for routine data maintenance.

## Proposed Changes

### 1. AI Rulebook Management
- **Commands**:
    - `/rulebook list` - List fine-tune examples and knowledge chunks.
    - `/rulebook add <type> <text>` - Add a new entry (Interactive flow).
    - `/rulebook delete <id>` - Remove an entry.
- **Implementation**: Add handlers for `LLMFineTuneExample` and `LLMKnowledgeChunk` models in `src/bot.py`.

### 2. Category CRUD
- **Commands**:
    - `/categories edit <id>` - Interactive prompt to change name, type, or parent.
    - `/categories delete <id>` - Soft/Hard delete confirmation.
- **Implementation**: Extend `categories_command` with subcommands.

### 3. Transaction Management
- **Search**: `/search <query>` - Find transactions by description or date.
- **Edit**: Inline buttons on transaction search results to "Edit Category", "Edit Amount", or "Delete".

## Verification Plan
1. Use `/rulebook list` to see existing data.
2. Add a test rule and verify it appears in results.
3. Edit a category name and verify in database.
4. Search for a transaction and use inline buttons to modify it.
