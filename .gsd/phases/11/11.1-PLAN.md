# Phase 11 Plan: Conversational Intelligence

Transition the bot interaction model from explicit commands to natural language intent handling.

## User Review Required
> [!IMPORTANT]
> The bot will now prioritize natural language processing for all messages. While commands like `/add` or `/categories` will still work, the primary way to interact will be through natural conversation.

## Proposed Changes

### Intent Classification Layer
#### [NEW] [intents.py](file:///Users/ajith/Documents/Bluecoins-Manager/src/intents.py)
- Implement `IntentAI` class using Ollama.
- Define a structured prompt to classify user messages into:
    - `ADD_TRANSACTION`, `ADD_ACCOUNT`, `ADD_CATEGORY`
    - `LIST_ACCOUNTS`, `LIST_CATEGORIES`, `LIST_RULEBOOK`
    - `REVIEW_QUEUE`, `MODIFY_TRANSACTION`
    - `CHAT_QUERY` (Fallback for RAG)
- Extract entities like `amount`, `description`, `account_name`, `category_name`, `transaction_id`.

### Bot Integration
#### [MODIFY] [bot.py](file:///Users/ajith/Documents/Bluecoins-Manager/src/bot.py)
- **`handle_chat_message`**:
    - Call `IntentAI.classify(message.text)`.
    - If it's an action intent:
        - Dispatch to specialized NL handlers.
        - e.g., if `ADD_ACCOUNT` is detected with entity `Wise`, start the account add flow and pre-fill the name.
    - If it's `CHAT_QUERY`:
        - Continue with existing RAG/Coaching flow.
- **Refactor Action Handlers**:
    - Update `start_add`, `acc_add_start_callback`, etc., to optionally accept pre-extracted data from the AI.

## Verification Plan

### Automated Tests
- Scripted tests with various phrases to verify correct intent classification:
    - "I want to add 50 for lunch" -> `ADD_TRANSACTION`
    - "Show my categories" -> `LIST_CATEGORIES`
    - "Modify transaction 12" -> `MODIFY_TRANSACTION`

### Manual Verification
- Conversational walkthrough:
    1. Send "I want to add Wise account".
    2. Check if bot skips step 1 (name) and asks for institution.
    3. Send "Modify transaction 10".
    4. Verify bot shows the edit menu for that specific transaction.
