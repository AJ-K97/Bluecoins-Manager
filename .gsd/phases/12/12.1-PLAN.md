# Plan 12.1: Testing Infrastructure & Synthetic Data

## Objective
Establish a robust testing environment that mocks external dependencies (Ollama, Telegram) and generates realistic, anonymized financial datasets for comprehensive verification.

## Context
- spec.md
- .gsd/ARCHITECTURE.md
- tests/ (existing tests)
- src/local_llm.py (Ollama dependency)
- src/bot.py (Telegram dependency)

## Tasks

<task type="auto">
  <name>Setup Mocking Infrastructure</name>
  <files>tests/conftest.py, tests/mocks.py</files>
  <action>
    - Create a centralized `conftest.py` for shared pytest fixtures.
    - Implement an `AsyncLocalLLMPipeline` mock that uses static JSON responses to simulate Ollama.
    - Setup a transient PostgreSQL database fixture using `pytest-asyncio` and `SQLAlchemy` (leveraging existing `AsyncSessionLocal`).
    - Mock `httpx.AsyncClient` calls to Ollama if direct mocking of the pipeline is insufficient.
  </action>
  <verify>pytest tests/test_infra.py</verify>
  <done>
    - [ ] Mock LLM returns predictable responses based on prompt keywords.
    - [ ] Database fixture provides a clean, schema-ready session for each test.
  </done>
</task>

<task type="auto">
  <name>Generate Synthetic Financial Data</name>
  <files>tests/data_gen.py, tests/input_data/synthetic_hsbc.csv, tests/input_data/synthetic_wise.csv</files>
  <action>
    - Build a script to generate "real-world" style CSVs for HSBC and Wise.
    - Include scenarios: multi-currency transactions, refunds, duplicate descriptions with different amounts, and various date formats.
    - Generate synthetic "Knowledge Chunks" and "Fine-tune Examples" for the AI rulebook tests.
  </action>
  <verify>python3 tests/data_gen.py --verify</verify>
  <done>
    - [ ] Synthetic CSVs exist and follow the exact column structure of real exports.
    - [ ] Dataset covers at least 10 diverse transaction scenarios.
  </done>
</task>

## Success Criteria
- [ ] Pytest can run without an active Ollama server.
- [ ] Database tests are isolated and don't require a persistent PostgreSQL instance.
- [ ] Synthetic data is generated and ready for parser/logic testing.
