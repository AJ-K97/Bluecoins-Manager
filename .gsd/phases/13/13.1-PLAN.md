# Plan 13.1: Schema Migration for AI Memory Persistence

## Objective
Update the database schema and ORM models to ensure that deleting a transaction does not delete its corresponding AI memory entries or fine-tune examples. We will implement "Soft Detachment" where the foreign keys are nulled out instead of cascading the delete.

## Context
- .gsd/SPEC.md
- src/database.py

## Tasks

<task type="auto">
