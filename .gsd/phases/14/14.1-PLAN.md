# Plan 14.1: Interaction Logging & Audit Trail

## Objective
Implement a comprehensive logging system to record user interactions, intent classification results, and bot actions. This will allow us to debug why specific queries (like "transactions for Feb 2026") are misclassified or mishandled.

## Context
- src/database.py
- src/bot.py

## Tasks

<task type="auto">
  <name>Create InteractionLog Schema</name>
  <files>src/database.py</files>
  <action>
    - Define `InteractionLog` model in `src/database.py`.
    - Columns: 
      - `id` (Integer, PK)
      - `timestamp` (DateTime, index=True)
      - `user_id` (Integer, index=True)
      - `username` (String, nullable=True)
      - `message_content` (Text)
      - `detected_intent` (String, nullable=True)
      - `confidence_score` (Float, nullable=True)
      - `entities_json` (Text, nullable=True)
      - `action_taken` (String, nullable=True)
      - `response_content` (Text, nullable=True)
  </action>
  <verify>
    grep "class InteractionLog" src/database.py
  </verify>
  <done>
    Schema defined in `src/database.py`.
  </done>
</task>

<task type="auto">
  <name>Implement Logging Service</name>
  <files>src/logger.py, src/bot.py</files>
  <action>
    - Create `src/logger.py` with an async function `log_interaction(session, user_id, content, intent, confidence, action, response=None)`.
    - Integrate this logger into `src/bot.py`:
      - Inside `handle_text_message` (log user input + detected intent + action routed).
      - Inside `handle_command` (log slash command + action).
  </action>
  <verify>
    grep "log_interaction" src/bot.py
  </verify>
  <done>
    Bot writes to `interaction_logs` table for every significant user action.
  </done>
</task>

## Success Criteria
- [ ] User messages are saved to `interaction_logs`.
- [ ] Intent classification (intent + confidence) is recorded for debugging.
