# Plan 5.1: Database Migrations (Alembic)

## Objective
Initialize Alembic to manage database schema changes, ensuring long-term stability for the persistent database. Baseline the current schema state.

## Context
- `src/database.py` (SQLAlchemy models)
- `docker-compose.yml` (Postgres container)

## Tasks

<task type="auto">
  <name>Initialize Alembic</name>
  <files>alembic.ini, migrations/env.py</files>
  <action>
    - Run `alembic init migrations`
    - Configure `alembic.ini` to use `DATABASE_URL` from env (via python-dotenv or expansion)
    - Update `migrations/env.py` to import `src.database.Base` for autogenerate support
    - Ensure async support in env.py (using `async_engine_connect`)
  </action>
  <verify>alembic check</verify>
  <done>Alembic initialized and configured</done>
</task>

<task type="auto">
  <name>Baseline Schema</name>
  <files>migrations/versions/*</files>
  <action>
    - Generate initial migration for existing schema: `alembic revision --autogenerate -m "initial_baseline"`
    - Verify the generated script matches `src.database` models
  </action>
  <verify>alembic upgrade head</verify>
  <done>Initial migration created and applied (stamping current DB)</done>
</task>

## Success Criteria
- [ ] `migrations` directory exists
- [ ] `alembic upgrade head` runs without error
- [ ] Future schema changes can be autogenerated
