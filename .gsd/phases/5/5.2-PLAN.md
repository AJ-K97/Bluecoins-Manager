# Plan 5.2: Access & Backup

## Objective
Ensure the database is truly "persistent" by implementing automated backups and documenting remote access for multi-machine usage.

## Context
- `docker-compose.yml`

## Tasks

<task type="auto">
  <name>Create Backup Script</name>
  <files>scripts/backup_db.sh</files>
  <action>
    - Create `scripts/backup_db.sh`
    - Use `docker exec` to run `pg_dump` inside the container
    - Save to `backups/` directory with timestamp
    - Add retention policy (keep last 7 days) if easy, otherwise just dump
    - Ensure script is executable
  </action>
  <verify>./scripts/backup_db.sh && ls backups/*.sql</verify>
  <done>Backup script creates valid SQL dump</done>
</task>

<task type="auto">
  <name>Document Remote Access</name>
  <files>docs/REMOTE_ACCESS.md</files>
  <action>
    - Create documentation on how to connect to this DB from another machine on the LAN.
    - Explain how to find LAN IP (`ip addr`).
    - Explain `DATABASE_URL` format for remote clients.
    - Verify default Postgres container explicitly allows remote connections (it usually does).
  </action>
  <verify>test -f docs/REMOTE_ACCESS.md</verify>
  <done>Documentation exists</done>
</task>

## Success Criteria
- [ ] Database backup created successfully
- [ ] Instructions for remote access are clear and written
