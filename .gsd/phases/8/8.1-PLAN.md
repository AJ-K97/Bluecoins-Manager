# Phase 8.1: Cloud Database Migration Plan

## Objective
Establish a persistent, cloud-hosted PostgreSQL database to allow cross-device access (Local CLI, Remote Bot) without home-hosting.

## Proposed Providers
- **Supabase** (Recommended): Free tier includes 500MB DB, easy UI, Postgres-native.
- **Neon**: Serverless Postgres, great free tier with 0.5 GiB storage.

## Migration Steps
1. **Provision Resource**: Create a new project on the chosen provider.
2. **Schema Migration**:
   - Run `alembic upgrade head` using the new `DATABASE_URL`.
   - Alternatively, use `python3 main.py db reset` logic to initialize.
3. **Data Migration**:
   - If home DB is inaccessible, we will seed from `data/accounts.json` and `data/categories.json`.
   - If home DB can be temporarily sparkled, we can perform a `pg_dump` and `pg_restore`.
4. **Update Environment**:
   - Update `.env` with the new cloud `DATABASE_URL`.
5. **Verification**:
   - Run `python3 check_db.py` to confirm connection.
   - Run the bot and verify Phase 7 commands work.
