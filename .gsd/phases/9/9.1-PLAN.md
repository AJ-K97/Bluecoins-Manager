# Phase 9.1: Bot UX & Discoverability Plan

## Objective
Enhance the Telegram bot's user experience by making commands discoverable, providing a central help guide, and improving message aesthetics.

## Proposed Changes

### 1. Command Registration (Menu)
- Use `context.bot.set_my_commands` on startup to register the primary commands:
    - `start` - Reset/Welcome
    - `help` - Show all commands
    - `stats` - Review queue status
    - `review` - Start interactive review
    - `accounts` - List accounts
    - `categories` - List categories
    - `add` - Add manual transaction

### 2. Help Command (`/help`)
- Create a dedicated `/help` handler.
- Output a structured message with:
    - **Quick Actions**: Buttons or commands for common tasks.
    - **Detailed Command List**: Descriptions and examples (e.g., `/add 10 Coffee`).
    - **Contact/System Info**: Link to docs or local LLM status.

### 3. Aesthetic Improvements
- **Templating**: Standardize formatting using specific emojis for different message types:
    - 📁 Accounts
    - 🏷️ Categories
    - 💰 Transactions
    - 🤖 AI Insights
- **MarkdownV2**: Transition more messages to MarkdownV2 (or keep Markdown but be more consistent) for bolding and code blocks.
- **Bot Menu (Reply Keyboard)**: On `/start`, show a `ReplyKeyboardMarkup` with "📊 Stats", "📝 Review", and "➕ Add" to reduce the need for typing.

## Verification Plan
1. Start the bot and verify the "Menu" button appears in Telegram.
2. Run `/help` and check formatting.
3. Trigger `/stats` and `/accounts` to verify new "prettified" layouts.
